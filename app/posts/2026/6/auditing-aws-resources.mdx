### Research

Let's take a look at some documentation for boto3 and look at the EC2 client. What I want to take a look at particularly is a method called describe_instances.

Here's a snippet of how we can use this method.

```python
client = boto3.client('ec2')
response = client.describe_instances(
    InstanceIds=['string',], 
    IncludeManagedResources=True|False,
    DryRun=True|False,
    Filters=[
        {
            'Name': 'string',
            'Values': [
                'string',
            ]
        },
    ],
    NextToken='string',
    MaxResults=123
)

```
As you can see, we can use the API to get really detailed on what kind of EC2 instances we want to look for. InstanceIds allows us to pass a list of strings to get all kinds of information we need about those EC2 instances, by default you query all instances.

We can also use filters, which is a list of hash tables with key-value pairs for almost anything you could think of on an EC2 instance. 

The other stuff isn't too important.

For this method, they do recommend the use of a paginator just because of how much data can potentially be pulled from AWS using this command. Let's follow their recommendation and also use their paginator.

### Implementation

Using what we just found in the boto3 documentation, let's build a python script that would search for a specific tag called "production" with the value "false" across all of my EC2 instances.

```python
import boto3

client = boto3.client('ec2')
paginator = client.get_paginator('describe_instances')
page_iterator = paginator.paginate(
    Filters=[
        {
            'Name': 'tag:production',
            'Values': [
                'false',
            ]
        }
    ]
)

instances = []
for page in page_iterator:
    for reservation in page['Reservations']:
        for instance in reservation['Instances']:
            instances.append(instance['InstanceId'])

print(instances)
```

We implemented the paginator, and we're getting a list of instance IDs that have the production tag with a value of false.

Now, we could have this ran serverless with a Lambda and tie it to an SNS so we can get emails of the JSON response. Just need to adjust the code a bit to accomodate this change.

```python
import json
import boto3

SNS_TOPIC_ARN = 'arn:aws:sns:us-east-1:123456789:audit-json'

ec2 = boto3.client('ec2')
sns = boto3.client('sns')


def lambda_handler(event, context):
    paginator = ec2.get_paginator('describe_instances')
    page_iterator = paginator.paginate(
        Filters=[
            {
                'Name': 'tag:production',
                'Values': [
                    'false',
                ]
            }
        ]
    )

    instances = []
    for page in page_iterator:
        for reservation in page['Reservations']:
            for instance in reservation['Instances']:
                instances.append(instance)

    instances_json = json.dumps(instances, indent=2, default=str)

    sns.publish(
        TopicArn=SNS_TOPIC_ARN,
        Subject='EC2 Non-Production Instance Audit',
        Message=instances_json,
    )

    return {
        'statusCode': 200,
        'body': instances_json,
    }
```
